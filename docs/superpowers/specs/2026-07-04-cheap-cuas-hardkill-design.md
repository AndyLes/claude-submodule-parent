# Дешевий стаціонарний hard-kill комплекс протидії БпЛА — Design Spec

**Дата:** 2026-07-04 (ревізія 2 — cost-exchange + повна автономність)
**Статус:** затверджено (архітектура B, фазове виконання; денний-перший Phase 1; ефектори зіставлені за вартістю; без пілотування)
**Дослідницька база:** [`2026-07-04-cheap-cuas-hardkill-research.md`](2026-07-04-cheap-cuas-hardkill-research.md) (13 агентів, 43 джерела, верифікація ключових цифр)

## 1. Мета та рамки

Спроєктувати **недорогий стаціонарний комплекс фізичного (hard-kill) перехоплення** ворожих БпЛА для захисту фіксованого об'єкта, з бюджетом **≤ $5 000 на весь сайт разом із першими ефекторами**, і — **головна ціль** — з **вартістю ураження нижчою за вартість цілі для противника**.

**Цілі (threat model):**
- (a) FPV-камікадзе;
- (b) розвідувальні квадрокоптери (DJI Mavic / Autel клас);
- (c) малі БпЛА л/к (Orlan-10 / ZALA / Supercam клас).

**Механізм:** hard-kill. **Soft-kill/РЕБ — поза scope** (волокно, AI-термінал, IMU, CRPA роблять глушіння малопридатним).

**Non-goals (Phase 1):**
- Не будуємо радар (від $30k).
- Не глушимо.
- Не покриваємо Орлан/ZALA на робочій висоті 1–1,5 км (Phase 3 — фікс-крило).
- **Немає ручного пілотування** — усе автономне; людина лише погоджує/скасовує.
- Не автономне ураження без дозволу людини.

## 2. Центральний принцип — обмін вартістю

«Наддешево + надефективно» знімається **двома важелями**, а не однією залізкою:
1. **Багаторазовість** — вартість платформи амортизується на десятки перехоплень; витрата/постріл = сітка/набій ($5–50), а не весь дрон.
2. **Автоматизація замість праці** — прибираємо пілота (кваліфікована праця за кожним пуском). Багаторазовість + автономність компаундяться: авто-дрон, що промахнувся, повертається й пробує ще за ~$0.

І — **зіставляти ціну ефектора з цінністю цілі:**

| Ціль | Ціна ворога | Одноразовий ~$500 | Обраний ефектор | Обмін |
|---|---|---|---|---|
| FPV (базовий) | $300–500 | ~1:1 ❌ | **E2** авто-таран ~$150–200 | 1,5–3:1 ✅ |
| FPV (волоконний) | $1 000+ | ~2:1 | **E2** ~$150–200 | 5–7:1 ✅ |
| Розвід-квадро (Mavic) | $2 000–4 000 | 4–8:1 | **E1** сітка ~$30/постріл | 60–130:1 ✅ |
| Орлан/ZALA/Supercam | $87k–120k | 170–240:1 ✅ | **E3** одноразовий (Phase 3) | ✅✅ |

Ніколи не витрачаємо $500 на $300-й FPV; дорогий одноразовий приберігаємо лише для $100k Орлана (E3, Phase 3).

## 4. Архітектура та потік даних

Ешелон **DETECT → CUE → AUTO-SOLVE → [OPERATOR OK] → LAUNCH → AUTO-TERMINAL → HIT**:

```
Загрози ─► [Акустика ×3] ┐
          [РЧ / SDR]     ├─► C2/ФУЗІЯ 2-з-N ─► трек+класифік. ─► АВТО-вогневе рішення ─► [ОПЕРАТОР: OK/ABORT] ─┐
          [EO/IR PTZ+AI] ┘        (slew-to-cue наводить PTZ на пеленг)                                        │
                                                                                                              ▼
   HIT ◄── авто-термінал (бортове машинне бачення + автопілот) ◄── E1 сітка-дрон / E2 авто-таран ◄── авто-ПУСК
```

**Потік:**
1. Акустика/РЧ → дешевий широкий **перший cue** + грубий пеленг (волоконні FPV бачить лише акустика).
2. C2 автослюінгом наводить **PTZ EO/IR** на пеленг → підтвердження + класифікація (птах/квадро/FPV/л-к).
3. Фузія **2-з-N** → трек якості вогневого керування; **C2 автоматично рахує вогневе рішення** і обирає ефектор за класом цілі (E1 для квадро, E2 для FPV).
4. **Оператор** бачить рішення й дає **OK** (або **ABORT**); no-fire дуги блокують пуск у населені сектори.
5. Авто-пуск; **термінал повністю автономний** (бортове машинне бачення pixel-lock + автопілот). Оператор може скасувати атаку до удару.
6. Удар: E1 — сітка (ціль заплутана, дрон повертається); E2 — таран + малий заряд.

## 5. Підсистеми

### 5.1 Виявлення (find-and-track)

| Вузол | Роль | Компоненти | Інтерфейс | Залежності |
|---|---|---|---|---|
| **Акустика ×3** | РЧ-тихий перший cue + пеленг; нічне попередження | параб. мік + MEMS + USB-звук + Pi Zero 2W + edge-ML; щогла 6–10 м | пеленг+клас → C2 (WiFi/LoRa) | синхро-час для тріангуляції (≥3) |
| **РЧ / SDR** | пеленг DJI/аналог-FPV коли випромінюють | RTL-SDR v4 + 2,4/5,8 ГГц + directional-патч | пеленг+частота → C2 | сліпий до волоконних/автономних |
| **EO/IR PTZ + AI** | сенсор вогневого керування: підтвердження, класифікація, трек | швидка PTZ денна камера + Pi 5 + Hailo-8L + YOLO | трек (az/el) + клас → C2 | потребує cue; денний (ніч — Phase 2) |

Свідомо прийняте: без радара → коротше попередження, гірше в туман/дощ; вночі — лише акустичне попередження. Надійний EO/IR-трек дешевого стека: сотні м … ~1–2 км.

### 5.2 C2 / вогневе керування (автономне)

- **Фузія 2-з-N:** трек підтверджується ≥2 модальностями до пуску (проти хибних).
- **Класифікація + вибір ефектора:** YOLO класифікує ціль; C2 автоматично обирає E1 (квадро/повільне) чи E2 (FPV) і рахує вогневе рішення.
- **Оператор (human-in-the-loop):** єдина дія — **OK на пуск** або **ABORT атаки**; здатний зупинити до удару; ніколи не наводить на людей.
- **No-fire логіка:** задані азимутальні дуги (населені сектори) блокують пуск/наведення.
- **Залізо:** працює на Pi 5 / Jetson + планшет-консоль оператора (без FPV-окулярів).

### 5.3 Ефектори (зіставлені за вартістю, повністю автономні)

- **E1 · Багаторазовий авто-дрон із сіткою** — головний проти розвід-квадрокоптерів і повільних лоітерів.
  - Планер (більший, несе сітколет + компут + гімбал-камеру), спусковий механізм сітки (пружина/CO₂), бортовий компут (Pi/Coral + камера) для авто-наведення, парашут/поворот на базу.
  - Ловить 4 відкриті ротори квадро → миттєва втрата тяги; **повертається**; витрата = картридж-сітка ($20–50) + знос.
  - Промах ≈ безкоштовно (повертається, пробує ще) — компенсує незрілість CV.
- **E2 · Наддешевий авто-таран (~$150–200)** — проти швидкого FPV на ближньому рубежі.
  - Стрипований 5″ автономний квадро: рама + мотори + FC(ArduPilot) + ELRS + **дешева камера + Pi Zero 2W/Coral** (бортове бачення), малий payload (таран/малий заряд). **Без VTX/пілота.**
  - Одноразовий, але дешевий → обмін 1,5–7:1 проти FPV.
- **E3 · (Phase 3) фікс-крило авто-інтерсептор** — проти Орлан/ZALA/Supercam (висота 1–1,5 км + дальність десятки км). Одноразовий, обмін 100:1.

**Пуск:** вертикальний із поста (E1/E2), без катапульти. **Термінал усіх ефекторів — бортове машинне бачення (pixel-lock) + автопілот**; оператор може скасувати.

*БЧ/заряд E2/E3 тут не деталізуємо — окрема інженерна/безпекова задача поза цим документом.*

## 6. Найкритичніша ланка — cue-handoff + автономний термінал

- Трек від сенсора має бути **суб-градусним**, щоб ціль потрапила у **вікно захоплення** борт-шукача (~500 м–2 км, ціль = кілька пікселів).
- PTZ має **точно слюінгувати під навантаженням** (0,1° втрати губить малий дрон).
- **Автономний термінал** замість пілота → успіх залежить від якості EO/IR + бортового CV. Ризик зрілості CV **пом'якшено багаторазовістю E1** (дешеві повторні спроби) + якісним трекером + ітеративним навчанням.
- Проти FPV критичні **латентність cue→пуск** і **маржа швидкості 25%** — **невиміряний параметр** → окрема задача моделювання (§10) до заяв про FPV-спроможність.

## 8. Фазовий роадмап

- **Phase 1 (≤$5k, цей документ):** пасивна детекція + автономний C2 (оператор OK/ABORT) + **E1 сітка-дрон** + **E2 ×4 авто-тарани**. Покриває розвід-квадрокоптери (E1, відмінний обмін) + FPV (E2, прийнятний обмін).
- **Phase 2 (+$1–3k):** low-res тепловізор (ніч), KrakenSDR (точний пеленг), покращений бортовий CV/трекер, +картриджі/+E2, друга E1-платформа.
- **Phase 3 (+$):** **E3 фікс-крило авто-інтерсептор** для Орлан/ZALA/Supercam; стандартний cue/track-інтерфейс для інтеграції в ППО.

---

## 9. Безпека та правові рамки

- **Людина в петлі** обов'язково: оператор дає OK/ABORT, може зупинити до удару; ніколи не автоцілити людей.
- **No-fire дуги** до населених секторів; **падіння уламків** невід'ємне → геометрія розміщення, безпечний фон. E1-сітка дає **низький збіг-збиток** (ціль опускається під сіткою).
- **Спектр:** сайт РЧ-залежний (детекція + керуючий лінк ефекторів); лінк частотно-гнучкий.
- **Правова заувага:** у США приватний kinetic hard-kill наразі незаконний без федерального партнерства (18 U.S.C. §32); застосування — лише в legally-authorized оборонному контексті.

## 10. Реалістичні очікування та ризики

**Очікування по цілях (Phase 1):**
- ✅ Розвідувальний квадрокоптер — **E1 сітка**, відмінний обмін (60–130:1), основний робочий кейс.
- 🟨 FPV-камікадзе — **E2 авто-таран**, прийнятний обмін (1,5–7:1); залежить від дальності детекції та зрілості бортового CV на швидкій цілі.
- 🟥 Орлан/ZALA/Supercam на висоті — поза Phase 1 (E3, Phase 3).

**Ключові ризики:**
1. **Cue-to-launch латентність + маржа 25% проти FPV** — невиміряно → окрема задача моделювання (обов'язкова перед заявами про FPV).
2. **Зрілість бортового CV без ручного резерву** — головна ставка на автономність; пом'якшено багаторазовістю E1 (дешеві повтори) + якісним EO/IR + тестуванням.
3. **Дальність детекції вночі** без тепловізора — лише акустика (короткий діапазон).
4. **False-alarm** одиничних сенсорів → фузія 2-з-N.
5. **E1 інженерна складність** (авто-наведення + сітколет + повернення) вища за простий таран — закладено як ризик-задачу в план.

## 11. Критерії приймання (acceptance)

- **Детекція:** кожен вузол ловить тест-ціль на заявленій дальності; фузія 2-з-N дає підтверджений трек із false-rate нижче цільового.
- **Cue-handoff:** PTZ автонаводиться на пеленг і утримує тест-ціль у полі захоплення на дальності ≥ X (визначити моделюванням §10).
- **C2 автономність:** система автоматично класифікує ціль, обирає ефектор і формує вогневе рішення; оператор одним підтвердженням дозволяє, ABORT зупиняє до удару.
- **E1:** авто-наведення на тест-квадро, скид сітки, захоплення, **повернення й повторне використання** відтворювані.
- **E2:** авто-таран досягає цільової швидкості й автономно вражає тест-мішень (без БЧ) під керуванням бортового бачення.
- **Безпека:** no-fire дуги блокують пуск у заборонені сектори; ABORT працює на будь-якій стадії.
- **Бюджет:** сумарний BOM сайту (з E1 + 4×E2) ≤ $5 000; підтверджено обмін вартістю на нашу користь по кожному класу цілей.
