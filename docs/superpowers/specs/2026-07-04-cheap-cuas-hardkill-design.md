# Дешевий стаціонарний hard-kill комплекс протидії БпЛА — Design Spec

**Дата:** 2026-07-04
**Статус:** затверджено (архітектура B, фазове виконання; денний-перший Phase 1)
**Дослідницька база:** [`2026-07-04-cheap-cuas-hardkill-research.md`](2026-07-04-cheap-cuas-hardkill-research.md) (13 агентів, 43 джерела, верифікація ключових цифр)

---

## 1. Мета та рамки

Спроєктувати **недорогий стаціонарний комплекс фізичного (hard-kill) перехоплення** ворожих БпЛА для захисту фіксованого об'єкта, з бюджетом **≤ $5 000 на весь сайт разом із першими інтерсепторами**.

**Цілі (threat model):**
- (a) FPV-камікадзе;
- (b) розвідувальні квадрокоптери (DJI Mavic / Autel клас);
- (c) малі БпЛА л/к (Orlan-10 / ZALA / Supercam клас).

**Механізм:** hard-kill (кінетичне перехоплення). **Soft-kill/РЕБ — поза scope** (волокно, AI-термінал, IMU, CRPA роблять глушіння малопридатним проти (a) і (c)).

**Non-goals (Phase 1):**
- Не будуємо радар (від $30k — поза бюджетом).
- Не глушимо (окремо незаконно/неефективно проти цілей).
- Не покриваємо повноцінно Орлан/ZALA на робочій висоті 1–1,5 км (це Phase 3 — фікс-крило інтерсептор).
- Не автономне ураження без людини (human-in-the-loop обов'язково).

## 3. Архітектура та потік даних

Ешелон **DETECT → CUE → DECIDE → LAUNCH → TERMINAL → HIT**:

```
Загрози ──► [Акустика ×3]  ┐
            [РЧ / SDR]     ├─► C2 / ФУЗІЯ 2-з-N ─► трек+класифік. ─► [ОПЕРАТОР] ─► Пуск ─► Інтерсептор ─► Термінал ─► Удар
            [EO/IR PTZ+AI] ┘        (slew-to-cue наводить PTZ на пеленг акустики/РЧ)         (ручний FPV / pixel-lock)
```

**Потік:**
1. Акустика/РЧ дають дешевий широкий **перший cue** + грубий пеленг (РЧ-тихі волоконні FPV бачить лише акустика).
2. C2 автослюінгом наводить **PTZ EO/IR** на пеленг → візуальне підтвердження + класифікація (птах vs дрон vs тип).
3. Фузія **2-з-N** підіймає трек до якості вогневого керування й давить хибні тривоги.
4. **Оператор** підтверджує ціль і дозвіл на пуск (no-fire дуги блокують пуск у населені сектори).
5. Пуск квадро-інтерсептора; наведення **ручним FPV** (база) або **pixel-lock AI** (Phase 2).
6. Удар: таран + малий заряд / проксіміті.

## 4. Підсистеми

### 4.1 Виявлення (find-and-track)

| Вузол | Роль | Компоненти | Інтерфейс | Залежності |
|---|---|---|---|---|
| **Акустика ×3** | РЧ-тихий перший cue + пеленг; нічне раннє попередження | параб. мік + MEMS-мік + USB-звук + Pi Zero 2W + edge-ML; щогла 6–10 м | пеленг+клас → C2 (WiFi/LoRa) | синхронізація часу для тріангуляції (≥3 вузли) |
| **РЧ / SDR** | пеленг DJI/аналог-FPV коли випромінюють | RTL-SDR v4 + 2,4/5,8 ГГц конвертер + directional-патч | пеленг+частота → C2 | **сліпий до волоконних FPV/автономних** |
| **EO/IR PTZ + AI** | сенсор вогневого керування: підтвердження, класифікація, трек | швидка PTZ денна камера + Pi 5 + Hailo-8L (або Jetson Orin Nano) + YOLO | трек (az/el) → C2 | потребує cue для наведення; денний (ніч — Phase 2 тепловізор) |

**Обмеження, прийняті свідомо:** без радара → коротше попередження, гірше в туман/дощ; вночі — лише акустичне попередження + денна камера безсила (до Phase 2). Дальність надійного EO/IR-треку дешевого стека — сотні метрів … ~1–2 км.

### 4.2 C2 / вогневе керування

- **Фузія 2-з-N:** трек підтверджується щонайменше двома модальностями (акустика+РЧ, або cue+EO/IR-підтвердження) до дозволу пуску — критично проти хибних (одиничний дешевий сенсор дає неприйнятний false-rate).
- **Класифікація:** YOLO на EO/IR відрізняє птах/дрон і по можливості клас (квадро vs л/к).
- **Оператор (human-in-the-loop):** підтверджує ціль, дає дозвіл; здатний зупинити в будь-який момент; ніколи не наводить на людей.
- **No-fire логіка:** задані азимутальні дуги/сектори (населені зони) блокують пуск/наведення туди.
- **Залізо:** працює на тому ж Pi 5 / Jetson (софт), + планшет/дисплей оператора.

### 4.3 Ефектор — квадро-інтерсептор

- **Планер:** 7–10″ рама, 4× високо-KV мотори + ESC, 6–8S LiPo (висока C), FC (ArduPilot/Betaflight) + ELRS RX, камера + VTX, малий payload-вузол.
- **Характеристики-ціль:** **250–350 км/год** (≥25% швидше за ціль), набір **~30 м/с**, стеля ≥ практичної для (a)/(b).
- **БЧ:** таран + малий заряд (удар / проксіміті / команда). *Специфікацію заряду тут не деталізуємо — окрема інженерна/безпекова задача поза цим документом.*
- **Пуск:** вертикальний із поста (без катапульти для квадро).
- **Термінал:** ручний FPV-пілот (база) → pixel-lock AI (Phase 2, ЕВ-стійкий: команда дається поза зоною глушіння).
- **Багаторазовість:** одноразовий; **боєкомплект = витратний**, поповнення поза $5k.

### 4.4 GCS (наземна станція) для ручного FPV
ELRS-передавач/пульт + FPV-окуляри + VRX. Може бути $0, якщо у оператора вже є FPV-комплект.

## 5. Найкритичніша ланка — cue-handoff

Найважче — **передача цілі**, не сам інтерсептор:
- Трек від сенсора має бути **суб-градусної точності**, щоб ціль потрапила у **вікно захоплення** борт-шукача (~500 м–2 км, ціль = кілька пікселів).
- PTZ-позиціонер має **точно слюінгувати під навантаженням** (втрата 0,1° на швидкому слюінгу губить малий дрон).
- Проти FPV критичні **латентність cue→пуск** і **маржа швидкості 25%** — це **невиміряний параметр**, який потрібно **промоделювати окремо** (див. §9 Ризики) до серйозних очікувань по FPV.

Сюди спрямовуємо основні інженерні зусилля Phase 1.

## 7. Фазовий роадмап

- **Phase 1 (≤$5k, цей документ):** пасивна детекція (акустика ×3 + РЧ/SDR + денний EO/IR+AI) + C2 з людиною в петлі + 4 квадро-інтерсептори + ручний FPV. Покриває розвід-квадрокоптери + низькі/повільні цілі; FPV — best-effort.
- **Phase 2 (+$1–3k):** low-res тепловізор (ніч), KrakenSDR (точний пеленг), **pixel-lock AI термінал** (ЕВ-стійкість, ×2–4 ефективності), більше інтерсепторів.
- **Phase 3 (+$):** **фікс-крило інтерсептор** для Орлан/ZALA/Supercam (висота 1–1,5 км + дальність ~десятки км); стандартний cue/track-інтерфейс для інтеграції в наявну ППО.

---

## 8. Безпека та правові рамки (частина конструкції)

- **Людина в петлі** обов'язково; оператор може зупинити; ніколи не автоцілити людей.
- **No-fire дуги** до населених секторів у fire-control-логіці; **падіння уламків** невід'ємне (снаряд + збита ціль падають) → геометрія розміщення, безпечний фон.
- **Спектр:** навіть hard-kill-only сайт РЧ-залежний (детекція + керуючий/відео лінк інтерсептора); лінк — частотно-гнучкий (противник контр-контрує детекторами інтерсептора).
- **Правова заувага:** у США приватний kinetic hard-kill наразі незаконний без федерального партнерства (18 U.S.C. §32); застосування — лише в legally-authorized оборонному контексті. *[research §6]*

## 9. Реалістичні очікування та ризики

**Очікування по цілях (Phase 1):**
- ✅ Розвідувальний квадрокоптер — основний робочий кейс.
- 🟨 FPV-камікадзе — можливо, але найжорсткіший (секунди на реакцію, маржа 25%); залежить від дальності детекції та вишколу пілота.
- 🟥 Орлан/ZALA/Supercam на висоті — здебільшого поза Phase 1 (потрібне фікс-крило).

**Ключові ризики:**
1. **Cue-to-launch латентність + маржа 25% проти FPV** — невиміряно → **окрема задача моделювання** (обов'язкова перед заявами про FPV-спроможність).
2. **Дальність детекції вночі** без тепловізора — покривається лише акустикою (короткий діапазон).
3. **False-alarm** одиничних сенсорів → пом'якшується фузією 2-з-N (планова робота ML/калібрування).
4. **Розрахунок/манінг 24-7** — хронічно недооцінена вартість; автоматизація cue + AI-термінал знижують навантаження (Phase 2).
5. **Вартість/УРАЖЕННЯ** реально ≈ $2–10k+ (часто >1 пуск/ціль) — інтерсептори витратні.

---

## 10. Критерії приймання (acceptance)

- **Детекція:** кожен вузол ловить тест-ціль на заявленій дальності; фузія 2-з-N видає підтверджений трек із false-rate нижче цільового.
- **Cue-handoff:** PTZ автонаводиться на пеленг і утримує тест-ціль у полі захоплення на дальності ≥ X (визначити моделюванням §9).
- **Ефектор:** інтерсептор досягає 250–350 км/год і набору ~30 м/с на випробуванні; ручний FPV-перехоплення тест-мішені (без БЧ) відтворюване.
- **Безпека:** no-fire дуги блокують пуск у заборонені сектори; оператор може перервати на будь-якій стадії.
- **Бюджет:** сумарний BOM сайту (з 4 інтерсепторами) ≤ $5 000.
