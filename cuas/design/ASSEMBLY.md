# E2 5″ авто-таран — друк і збірка

> Джерело чисел: [`e2_config.py`](e2_config.py) · ТТХ: `python -m cuas.design.report` · рама: [`cad/e2_frame.scad`](../../cad/e2_frame.scad).
> **Безпека:** ніс — інертна маса, **без БЧ**. Усі випробування — на інертних мішенях, у legally-authorized умовах.

## ТТХ (поточний конфіг)
AUW ~774 г · TWR ~7.0 · верт. приск. ~6 g · макс. ~168 км/год · політ ~3 хв · енергія тарана ~846 Дж · власна частота променя 35.8 Гц < збудження на висінні 202.6 Гц (без резонансу) · CoM +5.4 мм (злегка вперед — добре для тарана).

## Друк (PETG)
Рендер STL (потрібен OpenSCAD, безкоштовний):
```
mkdir -p cad/out
openscad -D 'part="frame"'        -o cad/out/e2_frame.stl        cad/e2_frame.scad
openscad -D 'part="battery_tray"' -o cad/out/e2_battery_tray.stl cad/e2_frame.scad
```
Налаштування слайсера (для міцності/одноразовості):
- Матеріал **PETG**, сопло 0.4, шар 0.2 мм.
- **Стінки: 4 периметри**; дно/верх 5 шарів.
- **Заповнення 35–40%**, gyroid (в'язкість під удар).
- Промені друкувати вздовж осі (міцність по довжині); без підпор, рама лежить дном.
- Орієнтація: рама пласко (дном на стіл), ніс — частина рами.

## Друк-лист
1. **Рама** (промені+хаб+ніс, цілісно) — ≤ ~180 мм, влазить у 220.
2. **Ложе акумулятора** (окремо).
3. *(Опц.)* носовий под під Pi+камеру — інтегрований у ніс (вікно cam_d).

## Порядок збірки
1. **Стек:** припаяти 4-в-1 ESC ↔ мотори (порядок фаз за схемою ArduCopter), FC на ESC (30.5), ELRS-RX. Перевірити на столі.
2. **Мотори:** прикрутити 2207 до майданчиків рами (M3×16 монтаж), пропи **знято** до кінця налаштування.
3. **Стек у ванну хаба** (30.5 M3, стійки); вивести кабелі.
4. **Ніс:** інертна маса + Pi Zero 2W з камерою у вікно носа (об'єктив вперед).
5. **Акумулятор:** 6S 1300 у ложе зверху, стрічка через прорізи.
6. **Баланс:** перевірити CoM — має бути біля центру, ледь вперед (±15 мм по x). Зсувати акум за потреби.

## ArduCopter (автономний термінал, per спека)
- Frame: **Quad X**; ESC-протокол DShot; калібрування мотор-порядку.
- **ELRS** RX; **Failsafe = RTL/Land** при втраті лінку (обов'язково).
- Компаньйон Pi/Coral → MAVLink GUIDED (наведення з `cuas/effectors/e2_ram/companion.py`).
- Політні тести: спершу висіння/стабілізація **без пейлоада й без пропелерного дисбалансу**, далі — швидкість/скоропідйомність, далі — авто-перехоплення інертної мішені.

## Чек перед польотом
- [ ] CoM у межах, пропи збалансовані, всі гвинти законтрені.
- [ ] Failsafe RTL спрацьовує (тест втрати лінку на землі).
- [ ] Резонанс: після 1-го висіння перевірити лог вібрацій (VIBE) ArduCopter; якщо високі — потовщити промінь (`ARM_H_MM` у конфізі → перегенерувати раму).
